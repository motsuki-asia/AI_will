## 1. プロジェクト概要

### 1.1 アプリ名

**AI will**

### 1.2 目的

音声で「恋愛・関係性（疑似恋愛/パートナー的関係）」から入り、ユーザーが"距離が縮まっていく体験"を継続的に楽しめる会話アプリを提供する。

あわせて、会話のレイテンシとAPIコスト制約を踏まえ、体験品質がブレない設計（自由会話＋イベント/定型フローのハイブリッド）で運用可能なMVPを構築する。

### 1.3 重要方針（MVP）

| 方針 | 詳細 |
|------|------|
| **プラットフォーム** | Web（PWA）先行。ネイティブアプリ（iOS/Android）は将来フェーズ |
| **クリエイター投稿** | MVPから実装。キャラクター作成・公開・収益化を可能にする |
| **収益化** | MVPから実装。フリーミアム＋キャラ課金（買い切り or チケット/クレジット制） |
| **本人音声** | 声素材/ボイスパックのアップロードをMVPから必須 |
| **レイテンシ対策** | 「相槌先出し＋正式回答」の2段階応答で体感待ち時間を改善 |
| **会話設計** | 自由会話（LLM生成）＋イベント会話（台本ベース）のハイブリッド |

### 1.4 スコープ

#### MVPでやること

- 音声会話（STT→LLM→TTS/ボイスパック）
- 好感度/親密度システム、関係ステージ
- イベント解放（告白、記念日等）
- クリエイターによるキャラクター作成・公開
- 本人音声アップロード（ボイスパック）
- 課金/収益分配（決済・分配基盤）
- 通報/モデレーション/公開停止
- 長期記憶（軽量版：呼び方、好み、NG、要約）
- 運営コンソール（最小限）

#### MVPでやらないこと（Phase 2以降）

- ネイティブアプリ（iOS/Android）
- カスタムTTS（本人ボイスでの自由会話生成）
- 高度なレコメンド/ランキングアルゴリズム
- コラボ/共同編集機能
- APIアクセス（外部連携）
- 多言語対応

## 2. ステークホルダー一覧

| 優先度 | ステークホルダー | 役割/説明 |
|--------|-----------------|-----------|
| **最優先** | エンドユーザー | キャラクターと音声会話を楽しむ利用者 |
| **最優先** | クリエイター | キャラクターを作成・公開する制作者（VTuber、声優、シナリオライター、イラストレーター等） |
| 高 | 運営者（事業責任者） | サービス全体の運営・事業判断を行う |
| 高 | 開発チーム | システムの設計・実装・保守を行う |
| 高 | 法規制/行政 | 個人情報保護法、特定商取引法、著作権法等を管轄 |
| 中 | モデレーター | コンテンツの審査・監視を行う |
| 中 | カスタマーサポート | ユーザー/クリエイターからの問い合わせ対応 |
| 中 | 外部サービス | STT/LLM/TTS、決済、クラウド等のAPI/インフラ提供者 |
| 低 | 投資家/出資者 | 資金提供者（将来フェーズで考慮） |
| 低 | メディア/社会 | 報道機関、SNS、一般社会 |

## 3. 機能要件

### 3.1 エンドユーザー向け

#### Must（必須）

---

##### EU-M01: 音声会話機能

| 項目 | 内容 |
|------|------|
| **目的** | キャラクターと音声でリアルタイムに会話し、自然なコミュニケーション体験を得る |

**受け入れ条件**
- [ ] 発話ボタンを押して話すと、音声認識（STT）でテキスト化され、AIが応答を生成し、音声（TTS/ボイスパック）で再生される
- [ ] 認識結果が字幕として表示され、誤認識時は「やり直し」ボタンで再発話できる
- [ ] 相槌先出し＋正式回答の2段階で応答される

---

##### EU-M02: 好感度/親密度システム

| 項目 | 内容 |
|------|------|
| **目的** | キャラクターとの関係が「進展している」実感を数値やビジュアルで確認する |

**受け入れ条件**
- [ ] 会話ごとに好感度/親密度パラメータが更新され、ユーザーに可視化される
- [ ] 好感度に応じてキャラの呼び方・口調・態度が段階的に変化する
- [ ] 「関係ステージ」（初対面→知り合い→友達→仲良し→恋人）を保持し、会話態度・解放イベントに反映される

---

##### EU-M03: イベント解放機能

| 項目 | 内容 |
|------|------|
| **目的** | 好感度が上がると特別なイベント（告白、記念日等）が解放され、達成感とドキドキ感を味わう |

**受け入れ条件**
- [ ] 好感度や条件を満たすとイベントが解放され、通知される
- [ ] イベント会話は台本ベースで進行し、ユーザーの選択で分岐する
- [ ] イベントに属性（1回きり/周回可/選択結果で状態変化）を持たせる
- [ ] イベント完了後、達成履歴が保存され振り返りできる

---

##### EU-M04: 通報/安全機能

| 項目 | 内容 |
|------|------|
| **目的** | 不快なコンテンツや問題のあるキャラクターを安全に報告し、安心して利用する |

**受け入れ条件**
- [ ] キャラクター/会話内容を通報できるボタンがあり、通報理由を選択/入力できる
- [ ] 通報後、運営による確認・対応フローが存在し、違反時は公開停止される
- [ ] 年齢確認（成人向けコンテンツがある場合）の導線が存在する

---

##### EU-M05: 課金導線（無料枠→有料）

| 項目 | 内容 |
|------|------|
| **目的** | 無料で試してから、気に入ったキャラやサービスに課金する |

**受け入れ条件**
- [ ] 無料枠が設定され、超過時に課金案内が表示される
- [ ] キャラ単位の購入、またはチケット/クレジット制で課金できる
- [ ] 決済基盤を通じて決済が完了すると即座に利用可能になる
- [ ] 購入処理は冪等性を担保し、二重課金を防止する
- [ ] 返金ポリシーが購入前に表示され、購入履歴で購入/解約/返金状態を確認できる

---

##### EU-M06: ユーザー認証/アカウント管理

| 項目 | 内容 |
|------|------|
| **目的** | 自分のデータ（会話履歴、好感度、購入履歴）を安全に保持し、継続利用する |

**受け入れ条件**
- [ ] メールアドレス or ソーシャルログインでアカウント作成/ログインできる
- [ ] ログイン後、会話履歴・好感度・購入履歴が復元される
- [ ] パスワードリセット、退会機能が存在する

---

##### EU-M07: 長期記憶（軽量版）

| 項目 | 内容 |
|------|------|
| **目的** | キャラクターに自分のことを覚えていてほしい（名前、好み、過去の会話） |

**受け入れ条件**
- [ ] ユーザーの呼び方（ニックネーム）、好み、NGワードが保存され、会話に反映される
- [ ] 過去の会話が要約され、長期記憶として保持される
- [ ] 関係性フラグ（告白済み、記念日設定等）が保持され、会話に反映される

---

#### Should（あると良い）

| ID | 機能名 | 目的 |
|-----|--------|------|
| EU-S01 | キャラクター検索/発見 | 好みのキャラクターを簡単に見つける |
| EU-S02 | お気に入り/フォロー機能 | 気に入ったキャラクターにすぐアクセスできる |
| EU-S03 | 会話履歴の閲覧 | 過去の会話を振り返る |
| EU-S04 | 字幕/テキストモード | 音声が使えない環境でも会話できる |
| EU-S05 | 利用時間通知（依存防止） | 長時間利用を自覚し、適度に休憩する |

---

#### Could（将来検討）

|-----|--------|------|
| EU-C01 | おすすめ/レコメンド機能 | 好みに合ったキャラクターを提案 |
| EU-C02 | プロフィールカスタマイズ | アバターやプロフィールを設定 |
| EU-C03 | 記念日リマインダー | キャラとの記念日を忘れずに祝う |

### 3.2 クリエイター向け

#### Must（必須）

---

##### CR-M01: キャラクター作成/公開

| 項目 | 内容 |
|------|------|
| **目的** | 自分のキャラクターを作成し、エンドユーザーに公開して利用してもらう |

**受け入れ条件**
- [ ] キャラクター基本情報（名前、画像、説明文、タグ）を入力できる
- [ ] 人格設定（性格、口調、世界観、禁止事項、サンプル会話）を入力できる
- [ ] 公開/非公開を切り替えられ、公開時は一覧に表示される

---

##### CR-M02: イベント台本作成（分岐対応）

| 項目 | 内容 |
|------|------|
| **目的** | 好感度に応じた特別イベントを台本形式で設計する |

**受け入れ条件**
- [ ] イベントの解放条件（好感度閾値、フラグ等）を設定できる
- [ ] ステートマシン形式でノード（セリフ）と分岐を定義できる
- [ ] 変数差し込み（ユーザー名、好み等）に対応している

---

##### CR-M03: 本人音声アップロード（ボイスパック＋権利同意）

| 項目 | 内容 |
|------|------|
| **目的** | 自分の声でキャラクターを喋らせ、オリジナリティを出す |

**受け入れ条件**
- [ ] 音声ファイル（挨拶、相槌、イベントセリフ等）をアップロードできる
- [ ] アップロード時に権利同意（本人/正当権利者であること、なりすまし禁止）への同意が必須
- [ ] アップロードした音声はイベント会話で優先再生される

---

##### CR-M04: 収益化（キャラ課金/チケット等）

| 項目 | 内容 |
|------|------|
| **目的** | 自分のキャラクターで収益を得る |

**受け入れ条件**
- [ ] キャラクターに価格（買い切り or チケット消費数）を設定できる
- [ ] 無料/有料の切り替えができる
- [ ] エンドユーザーが課金するとクリエイターの売上として計上される

---

##### CR-M05: 収益分配のための計測/帰属

| 項目 | 内容 |
|------|------|
| **目的** | 自分のキャラクターがどれだけ利用され、いくら収益が発生したか把握する |

**受け入れ条件**
- [ ] キャラクターIDごとに利用回数、売上が記録される
- [ ] ダッシュボードで売上サマリ、分配予定額を確認できる
- [ ] 分配ロジック（プラットフォーム手数料率等）が明示されている

---

##### CR-M06: モデレーション対応（通報対応/公開停止）

| 項目 | 内容 |
|------|------|
| **目的** | 通報や違反指摘があった場合、状況を把握し対応する |

**受け入れ条件**
- [ ] 自分のキャラクターへの通報があった場合、通知を受け取れる
- [ ] 違反により公開停止された場合、理由と異議申し立て方法が通知される
- [ ] 投稿ガイドラインが明示され、事前に確認できる

---

##### CR-M07: クリエイター認証/アカウント管理

| 項目 | 内容 |
|------|------|
| **目的** | クリエイターとしてログインし、自分のキャラクターや収益を管理する |

**受け入れ条件**
- [ ] クリエイター登録（メール or ソーシャルログイン）ができる
- [ ] 振込先情報（決済・分配基盤連携）を登録できる
- [ ] アカウント情報（表示名、プロフィール）を編集できる

---

##### CR-M08: 権利証跡・削除・異議申し立て

| 項目 | 内容 |
|------|------|
| **目的** | 自分の音声/コンテンツの権利を証明し、問題発生時に適切な対応を受ける |

**受け入れ条件**
- [ ] 音声アップロード時に権利者宣誓と証跡情報（氏名、連絡先、同意日時）が保存される
- [ ] クリエイターが自分の音声/キャラクターの削除申請ができる
- [ ] 運営による公開停止時、理由が通知され、異議申し立てフォームから申請できる

---

### 3.3 運営向け

#### OPS-M01: 運営コンソール（最小）

| 項目 | 内容 |
|------|------|
| **目的** | 運営者として、通報対応・違反コンテンツの確認・ユーザー管理を効率的に行う |

**受け入れ条件**
- [ ] 通報一覧を確認でき、ステータス（未対応/対応中/解決）で管理できる
- [ ] キャラクター/音声/台本の内容を閲覧し、公開停止/復帰操作ができる
- [ ] 権利証跡（アップロード時の宣誓情報）を参照できる
- [ ] ユーザー/クリエイターに対する制限（利用停止、投稿停止）を設定できる
- [ ] システムログ（会話ログ、課金ログ、操作ログ）を参照できる

#### 運営業務フロー（概要）

| 業務 | 対応内容 |
|------|----------|
| 通報対応 | 通報受付→内容確認→判定→公開停止 or 却下→通知 |
| 課金対応 | 購入確認→返金要求対応→チャージバック対応 |
| 監査 | 操作ログ/会話ログの閲覧、不正検知 |

## 4. 非機能要件

| ID | 要件名 | SLO | 測定方法 |
|-----|--------|-----|----------|
| **NFR-P01a** | 相槌/共感レイテンシ | 発話終了→相槌開始 **≤0.7秒**（P95）/ **≤1.2秒**（P99） | クライアント側タイムスタンプ計測 |
| **NFR-P01b** | 正式回答レイテンシ | 発話終了→正式回答開始 **≤3.0秒**（P95） | クライアント側タイムスタンプ、APM |
| **NFR-P02** | イベント/キャッシュレイテンシ | 入力→音声開始 **≤0.5秒**（P95） | クライアント側タイムスタンプ |
| **NFR-P03** | 音声アップロード処理時間 | 10MB以下 **≤30秒**（P95） | サーバーログ |
| **NFR-P04** | ページ初期表示速度（LCP） | **≤2.5秒**（P75） | Lighthouse / Web Vitals |
| **NFR-P05** | 同時接続数 | **≥100セッション**（MVP初期） | APMツール |
| **NFR-P09** | Fast Path安全制約 | 相槌は共感/確認/待機宣言のみ。不適切相槌 **0件** | 相槌候補リストの事前レビュー |
| **NFR-P10** | Fast→Slow切替制御 | 相槌は最大1回/2秒。2秒超→待機宣言+UI、5秒超→フォールバック | タイムスタンプ計測、ログ |
| **NFR-P11** | VAD/録音終了UI要件 | 手動停止ボタン必須、VAD閾値700ms、リカバリ手段提供 | UI監査、やり直し使用率 |

#### Should（あると良い）

| ID | 要件名 | SLO | 測定方法 |
|-----|--------|-----|----------|
| NFR-P01c | 正式回答完了時間 | 発話終了→正式回答完了 **≤8〜10秒**（P95）目安 | クライアント側タイムスタンプ |
| NFR-P06 | APIレスポンスタイム（非音声） | **≤500ms**（P95） | APMツール |
| NFR-P07 | TTS最初の音声チャンク | TTS開始→最初のチャンク **≤0.5秒**（P95） | サーバー/クライアント計測 |
| NFR-P08 | オートスケール応答時間 | 負荷増検知→新インスタンス稼働 **≤3分** | インフラ監視ツール |

### 4.2 セキュリティ/プライバシー

#### Must（必須）

| ID | 要件名 | SLO/要件 | 測定方法 |
|-----|--------|----------|----------|
| **NFR-S01** | 認証/認可 | 全APIで認証必須、脆弱性 **0件** | セキュリティテスト |
| **NFR-S02** | 通信暗号化（TLS） | **TLS 1.2以上**、非暗号化通信 **0件** | SSL Labs テスト |
| **NFR-S03** | データ暗号化（保存時） | PII/音声/証跡は **AES-256相当**、暗号化率 **100%** | インフラ設定監査 |
| **NFR-S04** | PII取り扱い | 最小権限原則、不正アクセス **0件** | アクセスログ監査 |
| **NFR-S05** | 音声データ保護 | 署名なしURL直接アクセス **0件** | ストレージアクセスログ |
| **NFR-S06** | 権利証跡保護 | 改ざん不可（書き込み専用ログ）、改ざん **0件** | ハッシュ検証 |
| **NFR-S07** | 鍵管理 | シークレットマネージャー使用、ハードコード **0件** | コードスキャン |

## 5. 用語集

| 用語 | 説明 |
|------|------|
| **Fast Path** | 相槌/共感など、事前生成済み音声を即座に再生するパス。低レイテンシを実現 |
| **Slow Path** | STT→LLM→TTSで正式回答を生成するパス。品質重視 |
| **TTFA** | Time To First Audio。発話終了から最初の音声再生までの時間 |
| **VAD** | Voice Activity Detection。発話の開始/終了を検知する技術 |
| **ボイスパック** | クリエイターがアップロードする本人声の音声ファイル集（相槌、イベントセリフ等） |
| **好感度** | キャラクターとの親密度を表す数値パラメータ |
| **関係ステージ** | 好感度に基づく関係性の段階（初対面→知り合い→友達→仲良し→恋人等） |
| **イベント** | 好感度や条件達成で解放される特別な会話シーン（告白、記念日等） |
| **ステートマシン** | イベント台本の分岐/遷移を管理するデータ構造 |
| **PWA** | Progressive Web App。ネイティブアプリに近い体験を提供するWebアプリ |
| **STT** | Speech-to-Text。音声をテキストに変換する技術 |
| **TTS** | Text-to-Speech。テキストを音声に変換する技術 |
| **LLM** | Large Language Model。大規模言語モデル |
| **冪等性** | 同じ操作を複数回実行しても結果が変わらない性質（二重課金防止等） |
| **決済・分配基盤** | クリエイターへの収益分配を可能にする決済プラットフォーム |
